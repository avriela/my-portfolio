Сервис сетевого доступа — бот и админ-панель
Сервис для продажи и управления подписками на сетевой доступ. Telegram-бот для пользователей и отдельная веб-админ-панель для управления сервисом. Несколько итераций тестирования — бот и админка как по отдельности так и в связке.
О продукте

Пользователь взаимодействует через Telegram-бот — выбирает план, оплачивает, получает ссылку для подключения к серверам. Доступно несколько серверов в разных локациях — при падении одного он автоматически скрывается из списка, остальные продолжают работать. При нестабильной работе сервера сервис переключается на резервный автоматически.

Несколько планов подключения с разным лимитом устройств и сроком действия. Пробный период — выдаётся один раз на аккаунт. Реферальная программа с начислением бонусов и выводом средств. Уведомления об истечении подписки, о начислениях и смене статуса. В личном кабинете — управление устройствами, история подключений, перевыпуск ключа.

Управление сервисом через отдельную веб-админ-панель: пользователи, подписки, платежи, рассылки, настройка текстов и кнопок бота.

Моя роль и задача

Проект проходил несколько итераций тестирования. Тестировала оба компонента — бот и админ-панель — как по отдельности так и в связке. Основной формат: составляла репорт со списком багов, передавала разработчику, после фикса проводила ретест.

Часть проблем уходила сразу в переписке — когда баг был очевидным и воспроизводился в одно действие. После каждого цикла фиксов часть багов выявлялась повторно уже на новых сценариях.

Что тестировала и как
Основные пользовательские сценарии

Полный путь нового пользователя — от первого запуска бота до активной подписки. Активация пробного периода и проверка что он выдаётся строго один раз на аккаунт. Выбор плана, оплата, получение ссылки для подключения.

Граничные сценарии — смена подписки до истечения текущей, поведение системы при наличии активной подписки при повторной покупке, первый запуск бота по реферальной ссылке.

Подключение и работа на устройствах

Корректность ссылки для подключения — формат, работоспособность, отображение нужных данных после вставки в приложение. Подключение с разных типов устройств — мобильные и десктоп.

Корректность счётчика подключённых устройств в реальном времени — увеличивается ли при подключении нового, уменьшается ли после удаления. Поведение при достижении лимита устройств по плану — блокируется ли новое подключение, корректно ли сообщение об ошибке.

Личный кабинет и управление устройствами

Корректность отображения данных пользователя — план, срок действия, количество устройств, история подключений. Добавление и удаление устройств, перевыпуск ключа и поведение старой ссылки после обновления — становится ли она недействительной.

Граничные случаи — попытка добавить устройство сверх лимита, состояние кабинета при отсутствии активной подписки.

Реферальная программа

Генерация реферальной ссылки и корректность перехода по ней. Начисление бонусов при выполнении условий, корректность суммы начисления для обеих сторон — пригласившего и приглашённого. Настройка реквизитов для вывода, запрос на вывод средств, история выводов.

Отображение статистики — количество приглашённых, количество оплативших, текущий баланс реферального счёта.

Уведомления

Уведомления об истечении подписки — приходят ли в нужный момент, корректен ли текст. Уведомления при ручных действиях через админку — начисление баланса, выдача дней подписки, изменение статуса.

Соответствие текста уведомления реальному действию, наличие причины если она была указана администратором. Отсутствие дублирующих уведомлений при повторных действиях.

Платёжная интеграция

Прохождение оплаты, автоматическая выдача доступа после успешного платежа. Оплата с бонусного баланса. Поведение при прерванной оплате — не выдаётся ли доступ при незавершённой транзакции. Корректность суммы к оплате при выборе разных планов. Отображение платежей в истории пользователя и в админке.

Защита от дублирования

Поведение при быстрых повторных нажатиях на кнопку оплаты — не создаётся ли дублирующая подписка. Повторная активация пробного периода с того же аккаунта. Поведение при одновременном открытии нескольких сессий бота.

Серверная логика

Отображение списка серверов после подключения. Поведение при недоступности одного из серверов — скрывается ли он из списка, продолжают ли работать остальные. Автоматическое переключение на резервный сервер при нестабильной работе.

Админ-панель — пользователи и подписки

Список пользователей и корректность отображения всех полей. Карточка пользователя — данные, история подписок, реферальная статистика, последние платежи. Ручное пополнение баланса и выдача дней подписки. Блокировка пользователя и её реальное влияние на доступ в боте. Карточка подписки — продление, деактивация, удаление. Поиск пользователя по ID и username.

Админ-панель — рассылки и управление контентом

Создание рассылки, фильтрация аудитории, предпросмотр перед отправкой. Счётчик результатов после отправки. Редактирование текстов бота, кнопок меню, инструкций, загрузка изображений.

Проверяла не только сохранение в админке но и реальное отображение изменений в боте — это отдельный важный шаг который легко пропустить.

Примеры найденных багов
Связка бот ↔ админка
  • Ручное начисление баланса через админку не давало возможности оплатить подписку этими средствами
  • Уведомления пользователю не приходили при начислении баланса, дней подписки и изменении статуса через админку
  • Текст причины при начислении не попадал в уведомление — приходило только стандартное сообщение
  • При выдаче дней подписки через админку дата в уведомлении пользователю была меньше чем до начисления
  • Блокировка пользователя не применялась — интерфейс сообщал об обратном действии, реального изменения статуса не происходило
  • Удаление подписки через админку не применялось — подписка оставалась активной несмотря на сообщение об успехе
  • Изменения внесённые в контент через админку не отображались в боте — ни тексты ни изображения
Данные и отображение
  • ID пользователей в списке обрезались — посмотреть или скопировать полный ID было невозможно
  • Карточка подписки открывалась с пустыми полями — данные не подтягивались
  • Активность пользователя в карточке не отображалась — постоянный прочерк
  • Реферальная статистика не подтягивалась ни в боте ни в админке
  • Статистика по результатам рассылки не считалась — счётчики показывали нули после отправки
Пользовательский бот
  • Ссылка для подключения не копировалась в буфер обмена в нескольких разделах — включая невозможность ручного выделения текста
  • Реферальная ссылка через кнопку "Поделиться" не работала — переход никуда не вёл
  • Количество привязанных устройств в личном кабинете отображалось как прочерк при наличии устройств
  • При попытке добавить второе устройство — ошибка без пояснения
  • Реферальный баланс показывал ноль при наличии средств на счету
  • Перевыпуск ключа не работал
  • Начисление бонусных дней рефереру происходило некорректно — выдавалось фиксированное значение вместо указанного в настройках
  • Ссылка в пробном периоде имела неверный формат — при этом в личном кабинете ссылка была корректной
  • Поиск пользователей в админке не работал
Результат

Протестированы оба компонента системы — бот и админ-панель — включая их взаимодействие в связке. Проект прошёл несколько итераций: после каждого цикла фиксов проводился ретест, часть багов выявлялась повторно на новых сценариях.

Выявлен и передан разработчику полный список проблем — от критичных (начисление средств не применялось, блокировка не работала, контент не обновлялся) до интерфейсных (обрезка ID, пустые поля в карточках, некорректные счётчики). После финального ретеста критичные баги устранены.
